# Visual QA Portfolio

## Quick Workflow

1. Start the app or use a static build preview.
2. Inspect desktop and mobile viewports.
3. Check relevant scroll states, not only the initial viewport.
4. Compare the result against `DESIGN.md`.
5. Validate reduced-motion and coarse pointer states when motion is involved.
6. Record precise tuning suggestions instead of vague visual feedback.

## Rules

- This portfolio uses craft as evidence; visual details matter.
- Preserve the cartographic paper and modern dashboard design direction.
- Avoid generic gimmicks that distract from the work.
- Ensure text, images, and controls do not overlap at common viewports.
- Use real browser inspection for layered image alignment.

## Avatar Eye Notes

- Check that eyes sit naturally within the glasses.
- Movement should be subtle and intentional.
- The effect should appear only in the interactive dashboard area.
- The static avatar state should still look finished.
